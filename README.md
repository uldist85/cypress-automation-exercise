# 🚀 Cypress UI & API Automation Project

This is a test automation project developed for the **Automation Exercise** platform. The repository contains both End-to-End User Interface (UI) tests implemented using the **Page Object Model (POM)** pattern and backend **API tests**.

The main goal of this project is to demonstrate modern test automation best practices, robust code architecture, and test stability using TypeScript.

---

## 🛠️ Tech Stack

* **Cypress** (Test Automation Framework)
* **TypeScript** (For strongly-typed, maintainable code)
* **Page Object Model (POM)** (Design pattern for scalable UI testing)
* **Git / GitHub** (Version Control)

---

## 📂 Project Structure

* `cypress/e2e/ui-tests/` – End-to-End (UI) test scripts (Login, registration, cart management, etc.).
* `cypress/e2e/api-tests/` – API-level validation scripts (Product search, login validation, negative testing).
* `cypress/page_objects/` – Page Object classes (Keeps UI selectors and actions clean and reusable).
* `cypress/fixtures/` – Static test data (User profiles and environments stored in JSON format).
